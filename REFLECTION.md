# Project Reflection

## Decisions

One of the biggest decisions I made during this project was choosing to create an educational IT and cybersecurity resource website instead of a more general website topic. I chose this direction because it connects to my interests, certifications, military IT experience, and long-term career goals. I also decided to focus on keeping the website beginner-friendly instead of making it overly technical. I wanted users to feel comfortable learning step by step without becoming overwhelmed.

Another important decision was improving the visual design and layout between Project 01 and Project 02. I focused on creating a cleaner interface with better spacing, stronger organization, images, responsive layouts, and improved navigation across the entire site.

---

## What Worked

The strongest part of my project was the overall organization and structure of the website. Each page focuses on a specific topic while still connecting together through the navigation system. I was especially satisfied with the certifications page because it included search and filtering functionality that made the website feel more interactive and professional.

The dark mode feature also worked well and improved the user experience. I learned how small design choices such as spacing, consistent colors, and organized layouts can significantly improve the appearance of a website.

---

## What I Would Do Differently

If I started the project again, I would spend more time planning the design and structure before writing code. Early in the project I changed layouts and sections multiple times, which created extra work later. I would also organize my GitHub commits more consistently instead of making large updates all at once.

I would also like to learn more advanced JavaScript and responsive design techniques so I could add even more interactivity and animations to the website.

---

## What I Learned

This project taught me a lot about how websites are built, organized, deployed, and improved over time. I learned how HTML, CSS, and JavaScript work together to create a functional website and how deployment through GitHub Pages works in a real project environment.

I also learned how important troubleshooting and iteration are in web development. Small mistakes in file paths, code structure, or styling can affect the entire project, so attention to detail is important.

Personally, I became more confident editing code, organizing project files, troubleshooting issues, and improving designs through practice and repetition. This project helped me better understand both technical web development skills and the overall development process.
