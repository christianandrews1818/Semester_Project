# Assignment 01: Project Proposal

## 1. Project Description

The project I will be building is an IT and Cybersecurity Resource Hub website designed to help beginners learn foundational concepts in information technology and cybersecurity. When a user visits the site, they will be able to navigate through organized sections that explain key topics such as networking basics, cybersecurity principles, and certification preparation. The experience will be simple, structured, and easy to follow for someone new to the field.

This project matters to me because many beginners struggle to find clear, beginner-friendly explanations of IT and cybersecurity topics. Much of the information online is either too advanced or scattered across different sources. This website aims to centralize that information and present it in a way that is easy to understand and accessible.

The intended audience includes college students, entry-level IT learners, and individuals interested in pursuing a career in cybersecurity or information technology. This also includes military personnel transitioning into IT-related careers who need a structured and straightforward learning resource.

---

## 2. Feature Plan

### Project 01 (Week 05 – Prototype)

- Homepage: A main landing page that introduces the website, its purpose, and its audience.
- Navigation Menu: A simple navigation system allowing users to move between sections of the site.
- IT Fundamentals Page: A page explaining basic IT concepts such as hardware, software, and networking.
- Cybersecurity Basics Page: A page introducing key cybersecurity concepts like threats, vulnerabilities, and protection methods.
- Clean Layout: A structured and readable layout using HTML and basic CSS.

### Project 02 (Week 08 – Final MVP)

- Certification Resources Page: A section dedicated to certifications like CompTIA A+, Network+, and Security+.
- Study Guides Section: Expanded content with more detailed explanations and structured learning paths.
- Interactive Features: Simple tools such as checklists or quizzes to help users test their knowledge.
- Improved Design: Enhanced styling using CSS to create a more professional and user-friendly interface.
- Additional Content: More topics and deeper explanations across all sections.

---

## 3. Tools and Technologies

- Code Editor: I will use Visual Studio Code because it is lightweight, widely used, and integrates well with extensions like Live Server for previewing changes.
  
- Framework/Approach: I will build the site using HTML and CSS from scratch. This allows me to fully understand the structure of web development without relying on frameworks, which is important for learning fundamentals.

- Hosting and Deployment: I will use GitHub and GitHub Pages to host and deploy the website. This allows for easy publishing and version control while making the site accessible through a public URL.

- AI Tools: I will use AI to assist or help generate code structure, troubleshoot issues, and improve content clarity. This helps speed up development while still allowing me to understand the code myself.

- Other Tools: GitHub Desktop will be used for version control and managing commits. This tool simplifies the process of tracking changes and pushing updates to GitHub.

---

## 4. Risks and Unknowns

One potential risk is implementing interactive features such as quizzes or dynamic elements, since I may need to learn basic JavaScript to make them functional. This is something I have limited experience with and may require additional time to learn.

Another risk is managing the scope of the project. It may be easy to try to include too many features or too much content, which could make it difficult to complete everything on time. To address this, I will focus on completing the core features first (Project 01) and then expand the project gradually.

A final unknown is improving the design and layout to look professional. While I can build the structure, creating a visually polished site may require additional practice with CSS styling.
